# Logic Analyzer Core

This document provides a highly detailed, step-by-step guide to installing, initializing, and running all components of the Logic Analyzer Core project. The system consists of four main parts: the FPGA logic, the MCU Bridge firmware, the Rust Backend data stream, and the Next.js Frontend dashboard.

## Prerequisites

Before starting, ensure your system has the following core dependencies installed:

1.  **PlatformIO IDE**: An extension for VS Code required to flash the Microcontroller (MCU).
2.  **Rust Toolchain**: Required to run the backend server. Install via standard Rust installation methods (e.g., rustup).
3.  **Node.js & npm**: Required to build and run the frontend web application (Node version 20+ recommended).

---

## 1. MCU Bridge Initialization (shrike-mcu-bridge)

The MCU Bridge acts as the interface between the FPGA and your computer. It requires flashing both the filesystem (to load the FPGA binary) and the actual firmware.

### Directory
Navigate to: `c:\logic-analyzer\shrike-mcu-bridge`

### Setup Instructions
1.  Open the `shrike-mcu-bridge` folder in VS Code with the PlatformIO extension installed. PlatformIO will automatically read the `platformio.ini` file and install the necessary dependencies, including the Raspberry Pi core and `shrike_arduino` library.
2.  **Upload the Filesystem Image**:
    *   Ensure the precompiled FPGA binary exists at `data/fpga.bin`.
    *   In the PlatformIO sidebar, go to "Project Tasks" -> "pico" -> "Platform" and click **Build Filesystem Image**.
    *   Once built, click **Upload Filesystem Image** while your MCU is connected to the computer. This writes `fpga.bin` to the LittleFS memory partition.
3.  **Upload the Firmware**:
    *   In PlatformIO, click the standard **Upload** button (usually a right-arrow icon in the bottom status bar).
    *   This compiles `src/main.cpp` and flashes the firmware to the board.
4.  **Verification**: 
    *   Open the PlatformIO Serial Monitor (baud rate 115200). 
    *   You should see debug messages confirming the LittleFS mount, successful FPGA flash, and a periodic heartbeat underscore (`_`) indicating it is running.

## 2. FPGA Component (logic-analyzer-fpga)

The Verilog code defining the signal generation is located in this directory. 

### Directory
Navigate to: `c:\logic-analyzer\logic-analyzer-fpga\shrike_logic_analyzer\ffpga`

### Usage Details
For standard operation, you do not need to interact with this folder. The compiled binary representing `src/main.v` is expected to already reside in the MCU bridge's `data` folder as `fpga.bin`.

If you modify the Verilog source code, you must re-synthesize it using your specific FPGA vendor toolchain, output a `.bin` file, rename it to `fpga.bin`, place it in the `shrike-mcu-bridge/data` directory, and repeat the **Upload Filesystem Image** step from Section 1.

## 3. Backend Setup (logic-analyzer-backend)

The backend is a high-speed data relay written in Rust. It captures Serial data from the MCU and broadcasts it over a WebSocket connection.

### Directory
Navigate to: `c:\logic-analyzer\logic-analyzer-backend`

### Configuration
1. Open `src/main.rs`.
2. Locate the line where the serial port is defined: `let builder = tokio_serial::new("COM9", 115_200);`
3. Update `"COM9"` to match the COM port assigned to your MCU device by Windows.

### Build and Run Instructions
1. Open a terminal in the `logic-analyzer-backend` directory.
2. Run the command:
   `cargo run`
3. The Rust package manager (Cargo) will fetch all dependencies (axum, tokio, etc.), compile the project, and execute the server.
4. You will see terminal output stating "Successfully hooked into Shrike Lite" and confirming the server is online at `http://127.0.0.1:4000`.

---

## 4. Frontend Setup (logic-analyzer-frontend)

The frontend visualizes the binary data streams coming from the backend using an HTML5 Canvas interface built with Next.js.

### Directory
Navigate to: `c:\logic-analyzer\logic-analyzer-frontend`

### Installation
1. Open a terminal in the `logic-analyzer-frontend` directory.
2. Download all Node module dependencies by running the command:
   `npm install`

### Run Instructions
1. In the same terminal, start the local development server:
   `npm run dev`
2. Open your web browser and navigate to `http://localhost:3000`.
3. The Web Application automatically connects to the backend WebSocket at `ws://127.0.0.1:4000/ws`.

## Final Operation Flow
To run the complete system reliably, ensure you boot components in the following order:
1. Ensure the MCU is plugged in and flashed correctly.
2. Start the Rust Backend (`cargo run`).
3. Start the Next.js Frontend (`npm run dev`) and open the browser. 

The frontend dashboard will immediately begin streaming and rendering the eight distinct logical waveforms generated by the hardware state.
